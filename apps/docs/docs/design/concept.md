# 设计理念

Forge Studio 的核心设计理念在于**化繁为简**，同时保持**高度的扩展性**。

## 1. 为什么从头开始？

随着 3D Web 应用的复杂性日益增加，我们发现现有的解决方案往往在“开箱即用”和“深度定制”之间难以达到平衡。因此，我们决定从零开始设计 Forge Studio。

## 2. 核心原则

- **架构优先**：采用 Monorepo 和组件化设计，确保每一层逻辑都能被解耦和单独测试。
- **生与产分离**：将系统划分为“生产者”(`@forge/editor`) 和“消费者”(`@forge/core`)，两者通过标准的 JSON 场景协议进行通信，实现编辑态和运行态的完全解耦。[了解更多详情](/design/architecture)
- **拥抱现代生态**：深度集成 Vue 3 的响应式系统和 Vite 的极速构建能力。
- **开发者体验 (DX)**：提供完善的类型推导（TypeScript）和开发者友好的调试工具。
- **面向未来**：在底层渲染引擎的抽象上预留足够的灵活性，以便在未来支持更高级的 WebGPU 特性。

> “好的架构不是设计出来的，而是演进出来的。但一个好的起点，能让演进的道路少走很多弯路。”
